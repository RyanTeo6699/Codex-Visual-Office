# Commercialization Pre-Scope Notes

## Purpose

Capture future commercialization options without implementing commercial features.

## Potential Future Commercialization Paths

- Local paid tool.
- Self-hosted package.
- Consulting delivery.
- Enterprise internal tool.
- Future cloud/team edition.

## Required Future Work

- License model.
- Support policy.
- Pricing.
- Update policy.
- Privacy/security statement.
- Terms and disclaimers.
- Public website or docs hub.
- Release artifact provenance.
- Customer onboarding and troubleshooting playbook.

## Commercial Readiness Gaps

- No signed/notarized installer.
- No auto updater.
- No public support policy.
- No license enforcement.
- No payment flow.
- No auth/account system.
- No cloud/team backend.
- No legal/privacy docs for public sale.

## Explicitly Not Implemented

- Payment.
- Auth.
- License enforcement.
- SaaS backend.
- Team workspace.
- Cloud sync.
- GitHub API.
- Vercel.
- Supabase.
- MCP / ChatGPT App.

## Recommended Timing

Commercialization should wait until at least one distribution path is validated with private local beta users and GM approves support, legal, update, and pricing scope.
