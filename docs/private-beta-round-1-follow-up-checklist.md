# Private Beta Round 1 Follow-Up Checklist

## Before Follow-Up

- Confirm tester ID.
- Confirm latest invitation/onboarding/submission status.
- Confirm whether support was previously requested.
- Check for duplicate follow-up.
- Prepare a short, local-first reminder.

## During Follow-Up

- Ask whether the tester still intends to participate.
- Ask whether setup is blocked.
- Ask whether support is needed.
- Ask for bounded, redacted error information only if needed.
- Confirm no secrets should be shared.

## After Follow-Up

- Update invitation status tracker.
- Update onboarding status tracker.
- Update submission status tracker.
- Update non-response tracker.
- Escalate blockers if real safety/data concerns are reported.

## Safety Reminder

Do not request:

- Tokens.
- `auth.json`.
- `.env`.
- `.env.local`.
- Private keys.
- Passwords.
- Credentials.
- Proprietary source files.
- Full logs.
- Full source archives.
- Local SQLite databases.

## How To Update Trackers

- Use only real tester replies, support observations, or GM manual notes.
- Keep placeholders as pending.
- Do not count non-response rows as feedback submissions.

## How To Escalate A Blocker

- Mark blocker in onboarding tracker.
- Create a sanitized support observation.
- Do not paste secrets.
- Notify GM if the blocker is P0/P1 or safety/data related.
