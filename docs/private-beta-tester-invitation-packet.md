# Private Beta Tester Invitation Packet

## Invitation Purpose

Invite selected testers to run Codex Visual Office as a local-first private beta and submit structured feedback.

This is not a public release, commercial launch, signed installer, notarized app, auto-updating product, cloud sync product, team workspace, MCP product, ChatGPT App, auth/payment product, GitHub API integration, Vercel integration, Supabase integration, or OpenAI API integration.

## What The Product Is

Codex Visual Office is a local-first visual workflow office for Codex-assisted project work. It visualizes projects, Codex seats, tasks, runner status, Git observation, quality gates, review decisions, local settings, backups, archive summaries, and safety boundaries.

## What This Beta Is Testing

- Source checkout setup clarity.
- Local launcher / browser workflow clarity.
- Local-first safety copy.
- Core routes:
  - `/`
  - `/settings`
  - `/safety`
  - `/archive`
  - `/projects/provider-workspace`
  - `/review/task-provider-review`
- Whether Review Room 2.0, Quality Gates, Git observation, Scope Guard, Settings, Backup/Restore, Archive, and Safety Room are understandable.
- Tester confusion, blockers, setup friction, and documentation gaps.

## Who This Beta Is For

- Technical testers comfortable with local source checkout.
- Users who can run a local Next.js app.
- Users who understand this is not a packaged public desktop installer.

## Current Limitations

- No signed installer.
- No notarized app.
- No auto updater.
- No cloud sync.
- No team workspace.
- No auth or payment.
- No MCP / ChatGPT App.
- No public support SLA.
- No production deploy.

## Required Prerequisites

- macOS development environment.
- Node and npm installed.
- Access to the private repository or provided source bundle.
- Ability to run local commands from documented setup instructions.
- Browser access to `http://localhost:3000`.

## Setup Expectation

Testers should follow the local setup guide and report where setup is unclear. They should not modify production credentials, upload source code, or share secrets.

## Safety Notice

Do not send:

- Tokens.
- `~/.codex/auth.json`.
- `.env` files.
- `.env.local` files.
- Private keys.
- Passwords.
- Credential-bearing logs.
- Proprietary source snippets.
- Full source archives.
- Local SQLite database files.

Use bounded, redacted excerpts only when a log or screenshot is necessary.

## Expected Time Commitment

Suggested beta session: 30-60 minutes.

## How To Submit Feedback

Submit feedback through the GM-approved private beta feedback channel using the feedback submission instructions. Use an anonymized tester ID if requested by GM.
